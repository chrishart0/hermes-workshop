# Workshop Backlog

A lightweight backlog for turning this repository from a workshop git repo into a polished 3–4 hour Hermes workshop experience.

## How to use this backlog

- Each ticket is a small Markdown file in [`tickets/`](tickets/).
- Keep tickets practical: goal, scope, acceptance criteria, and notes.
- Update status here as work moves.
- Prefer small commits that close one ticket at a time.
