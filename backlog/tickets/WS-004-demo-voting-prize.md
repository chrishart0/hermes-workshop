# WS-004 - Demo, voting, and prize format

## Status

Todo

## Priority

P2

## Goal

Design a simple final demo and voting format for the last 30 minutes of the workshop.

## Scope

- Decide demo length based on expected attendance.
- Define what attendees should show.
- Define voting categories.
- Keep it fun, low-pressure, and beginner-safe.
- Include practical logistics: HDMI table, timer, voting method, prize.

## Acceptance criteria

- [ ] Final 30-minute format is documented.
- [ ] Demo timing works for realistic room sizes.
- [ ] Voting criteria are simple.
- [ ] Format encourages rough-but-real demos, not polished perfection.
- [ ] Presenter guide links to the demo plan.

## Notes

Default framing: “Show us the useful loop you started, even if it is rough.”
