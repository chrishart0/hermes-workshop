## Core promise

Attendees should leave with a useful Hermes agent started or working - not just an abstract demo.

The core workshop goal is for each attendee to identify **their own useful agent idea** and implement a first working version or strong starting point. The Daily Intelligence Agent is only one sample path / teaching example, not the main promise of the event.

Examples of possible paths include daily intelligence, release/CVE watch, homelab or ops health, incident triage, ChatOps over local docs/data, personal admin automation, business reporting, or another recurring workflow the attendee actually wants automated.

## Audience

Good fits:

- people curious about practical AI agents
- technical professionals who want automation that actually runs
- local builders/operators/homelab people
- founders, analysts, organizers, marketers, and knowledge workers with repetitive information workflows
- programmers who want a better agent framework and workflow

The copy should not assume attendees are all programmers.

## What attendees bring

- Laptop
- Optional: OpenAI/Codex, OpenRouter, Anthropic, or another model/provider path
- Optional: Telegram/Discord/Slack/Teams delivery target if they want chat delivery
- One repetitive information task or workflow they would like to automate

## What the workshop covers

1. Art of the possible - what useful agents can do now
2. Hermes setup - install, configure provider/model, smoke test
3. Daily Intelligence Agent - shared baseline build
4. Custom build lab - adapt the pattern to a real workflow
5. Demos, voting, and prize

## Brand / copy direction

- Practical, not hypey.
- “Build your first useful AI agent” is the clearest broad headline.
- “Not another demo” is useful as a supporting phrase.
- Emphasize: hands-on, 4 hours, laptop, food included, leave with something working.
- Avoid jargon-heavy copy: explain Hermes as an open-source agent framework that can use models, tools, memory, skills, schedules, and chat delivery.
